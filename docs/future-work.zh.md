# 未來工作

ThinkingOS v1.0 已建立穩定契約。v1 之後的開發應深化證據、工具與互通性，同時維持驗證優先，且不與任何供應商綁定。

## 近期優先項目

- 發布跨 Adapter 比較正規化結果的一致性 Fixture。
- 為版本化知識資產增加來源與相容性 Metadata。
- 以 SDK 決定性契約提供參考 MCP Server。
- 建立技能與 Schema 版本升級工具。
- 擴大文件與範例的無障礙及在地化審查。

## 生態系優先項目

- 具有簽署來源 Metadata 的第三方技能套件探索。
- 不依賴中央執行平台的 Registry 聯邦。
- 相容開放可觀測標準的 Trace 與 Evaluation 匯出。
- 跨語言契約 Fixture 穩定後提供其他語言 SDK。
- 由 Core 之外獨立維護、以證據為基礎的領域套件。

## 明確非目標

- 選擇偏好的 AI 供應商或模型。
- 用自動分數隱藏不確定性。
- 集中保存使用者資料、憑證或私人對話狀態。
- 為了方便而讓一個技能涵蓋多種思考習慣。
- 將生成的解釋當成證據替代品。

優先順序可透過審查過的 Issue 與設計提案演進；破壞性變更保留給未來主版本。
