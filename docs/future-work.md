# Future Work

ThinkingOS v1.0 establishes stable contracts. Post-v1 development should deepen
evidence, tooling, and interoperability without weakening validation-first
behavior or coupling the framework to a provider.

## Near-term priorities

- Publish conformance fixtures that compare normalized results across adapters.
- Add provenance and compatibility metadata for versioned knowledge assets.
- Provide a reference MCP server around the SDK's deterministic contracts.
- Add migration tooling for Skill and Schema version upgrades.
- Expand accessibility and localization review for documentation and examples.

## Ecosystem priorities

- Third-party Skill package discovery with signed provenance metadata.
- Registry federation without a central execution platform.
- Trace and evaluation exports compatible with open observability standards.
- Additional language SDKs after cross-language contract fixtures stabilize.
- Evidence-backed domain packs maintained independently from the Core.

## Explicit non-goals

- Selecting a preferred AI provider or model.
- Hiding uncertainty behind automated scores.
- Centralizing user data, credentials, or private conversation state.
- Expanding a Skill to cover multiple thinking habits for convenience.
- Treating generated explanations as a substitute for evidence.

Priorities may evolve through reviewed issues and design proposals. Breaking
changes remain reserved for a future major release.
