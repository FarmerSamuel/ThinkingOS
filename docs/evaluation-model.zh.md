# ThinkingOS 評估模型

ThinkingOS 評估的是使用者推理品質，而不是智力、人格或價值。評估必須依據證據、保持透明，並配合現有資訊校準。

## 評估面向

### 輸入品質

判斷事實、目標、證據、假設、限制與定義是否相關、可信、清楚，且足以支援預定分析。

### 邏輯品質

判斷結論是否能由前提推出、因果主張是否有根據、是否考慮替代解釋，以及是否避免無證據跳躍。

### 完整性

判斷是否考慮重要輸入、利害關係人、限制、替代方案、風險與未解問題。完整性相對於已驗證目標，而不是分析篇幅。

### 一致性

判斷主張、假設、標準與建議是否相容，並在推理過程中保持穩定；合理變更必須明確說明。

### 可行動性

判斷推理是否支持清楚的決策、測試、問題或下一步，並具備足夠的責任、優先順序與限制。

## 分數與信心

實作可以使用文件化量尺評分，但必須同時提供優點、弱點、缺少資訊、邏輯問題與建議。信心代表評估可靠度，而非結論是否討喜；當證據不足、模糊或品質低時必須降低。各面向不可混為一談：完整論證仍可能邏輯薄弱，邏輯正確的論證也可能因關鍵輸入不足而無法行動。
