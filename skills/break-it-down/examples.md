# Break It Down Examples

## Simple case

Decompose “prepare a weekly meal” by process: select meals, acquire ingredients, prepare components, cook, store. Validate shared dependencies such as dietary constraints and storage capacity.

## Business case

Decompose a validated “slow order fulfillment” problem by process stage: order validation, inventory allocation, picking, packing, carrier handoff. Do not mix departments and process stages at the same level.

## Learning case

Decompose “explain hypothesis testing” into concepts: hypotheses, error types, significance, test selection, interpretation. Stop when each concept can be explained and tested independently.

## Failure case

“Marketing, poor software, training, and Q4” mixes responsibility, cause, intervention, and time. The skill must reject the mixed basis and request a coherent organizing principle.
