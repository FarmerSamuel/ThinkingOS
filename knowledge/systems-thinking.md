# Systems Thinking Knowledge

## System view

A system is a set of elements whose interactions produce behavior within a declared boundary. The boundary is an analytical choice, not an objective fact, and should name important interfaces and excluded concerns.

## Reusable concepts

- **Element:** A component with a role or state.
- **Interaction:** A transfer, influence, dependency, or constraint between elements.
- **Feedback loop:** A closed influence path that reinforces or counteracts change.
- **Delay:** A time gap that can obscure cause and effect.
- **Nonlinearity:** A relationship whose effect is not proportional to its input.
- **Adaptation:** A change in behavior in response to experience or environment.
- **Emergence:** A system-level pattern arising from interactions that isolated elements do not sufficiently explain.

## Analysis discipline

1. Validate the problem and goal.
2. Declare boundary, stakeholders, interfaces, and timeframe.
3. Decompose enough to identify relevant elements without destroying relationships.
4. Map dependencies, feedback, delays, and uncertainty.
5. Test alternative explanations before labeling behavior complex or emergent.

## Common errors

- Equating many parts with complexity.
- Treating correlation as interaction mechanism.
- Moving the boundary to exclude inconvenient effects.
- Explaining system behavior only through individual intent.
- Calling any surprising outcome emergent.
