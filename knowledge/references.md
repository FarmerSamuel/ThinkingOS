# Knowledge Base References

## Governance

ThinkingOS knowledge assets are reusable explanatory references. They are separate from skill logic and must include sufficient provenance for maintainers to review relevance and compatibility.

## External reference categories

Future sourced editions should prefer stable primary or authoritative works in:

- Decision theory and judgment under uncertainty.
- Scientific reasoning and hypothesis testing.
- Systems dynamics and complexity science.
- Problem structuring and operations research.
- Learning science, retrieval practice, and elaboration.
- Argumentation, informal logic, and critical thinking.

## Citation requirements

Every external entry should include author or responsible organization, title, publication or edition, year, stable identifier or URL when available, access date for mutable web sources, and a note explaining relevance. Sources inform reasoning but never override validation, user context, or declared uncertainty.

## Maintenance

Knowledge changes are reviewed independently from skill logic. Breaking terminology changes require a migration note, affected-skill review, and Semantic Versioning assessment.
